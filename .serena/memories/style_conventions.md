# Style & Conventions
- Use file-scoped namespaces (`namespace MimicXml;`), keep `using` directives grouped/sorted/deduped at the top, and favor `using` statements over fully-qualified names inside method bodies.
- Respect nullable annotations; guard before dereferencing optional values, prefer `var` when the RHS is obvious, otherwise use explicit types, and follow PascalCase for types/members and camelCase for locals/params with descriptive names.
- Favor guard clauses/early returns, deterministic output (`OrderBy` before emitting files), LINQ for transforms (fallback to loops when mutation/logging is necessary), and avoid async-over-sync patterns.
- Use object initializers for option/config setup, precise exception types (`ArgumentException`, `ArgumentNullException`, `FileNotFoundException`, `InvalidOperationException`), and log progress via `Core.Util.Logger.WriteLine` while honoring verbosity.
- DI: register services in `AppHost.CreateBaseServices`, resolve through `AppHost.GetService<T>()`, keep lifetimes consistent (mostly singleton), support `Func<ModificationAssignmentStrategy, IModificationAssignmentService>` factories, and launch external processes with `UseShellExecute=false`.
- Tests: NUnit style `Assert.That(...)`, use `[TestCase]` for parameterized tests, reference `TestContext.CurrentContext.TestDirectory` for fixtures, and clean up generated artifacts like `*_Entrapment.xml`.
