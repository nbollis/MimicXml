C# conventions in repo: file-scoped namespaces, nullable enabled, implicit usings enabled, guard-clause validation for IO/arguments, and dependency injection via Microsoft.Extensions.DependencyInjection. Use AppHost.CreateBaseServices + AppHost.GetService<T>() instead of direct construction when service lifetimes matter. Prefer deterministic ordering before writing outputs, keep logs via Core.Util.Logger and respect Verbose behavior. Common error types: ArgumentException for invalid args, FileNotFoundException for missing files, InvalidOperationException for unsupported states. Keep using directives tidy and remove unused imports. Tests use NUnit with Assert.That(...) style and TestContext.CurrentContext.TestDirectory for fixture paths.
